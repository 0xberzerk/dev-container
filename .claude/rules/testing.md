# Testing Patterns for Security Auditing

## POC Writing

POCs are targeted exploit tests for specific findings — not exhaustive branch coverage. Each POC proves a single vulnerability is exploitable.

### POC Structure

1. **Setup:** Deploy the vulnerable contract(s) in the state needed for the exploit
2. **Pre-conditions:** Assert the state before the attack (proves the starting point)
3. **Attack:** Execute the exploit steps
4. **Post-conditions:** Assert the attacker gained value, victim lost value, or invariant broke

### Fork Testing (Default)

Auditors almost always work against forked chains. Fork testing is the default, not the exception.

```solidity
// Pin to a specific block for deterministic replay
vm.createSelectFork(vm.envString('FORK_RPC_URL'), blockNumber);
```

- Always pin to a specific block number for reproducibility
- Use `vm.envString('FORK_RPC_URL')` — never hardcode RPC URLs
- Impersonate real accounts with `vm.prank()` / `vm.startPrank()`
- Label addresses for readable traces: `vm.label(address, "Uniswap Router")`

## Fuzz Testing

- Use `bound(input, min, max)` to constrain fuzz inputs — never `vm.assume()`
- Fuzz runs: 5,000 default, 10,000 CI
- Focus fuzz on state-changing and state-reading functions
- Pure functions are excluded (deterministic, no state to fuzz)

## Invariant Testing

- Handler contracts wrap target functions with bounded inputs
- Track expected state with `ghost_` variables
- Track call distribution with `calls_` counters
- Use `targetContract()` to restrict fuzzer scope
- Always produce both bounded and unbounded entry points:
  - Bounded: constrain inputs to realistic ranges (avoids noise)
  - Unbounded: explore full input space (catches edge cases)

## Test Directory Structure

```
test/
  Base.t.sol              # Global base (audit actors: attacker, victim, admin, protocolOwner)
  poc/                    # Targeted exploit tests for confirmed findings
  invariant/              # Protocol invariant tests
    handlers/             # Handler contracts wrapping targets
  fuzz/                   # Fuzz test suites (generated by Fuzz Agent or manual)
```

## Actors

| Actor | Role | Trust Level |
|---|---|---|
| `attacker` | Adversarial — tries to exploit | Untrusted |
| `victim` | Target of the exploit | Trusted user |
| `admin` | Protocol administrator | Privileged |
| `protocolOwner` | Highest privilege role | Highest |
